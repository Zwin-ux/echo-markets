# Contributing to Echo Markets

Thank you for your interest in contributing! Please follow these guidelines to help us maintain a high-quality, consistent codebase.

## Getting Started
- Fork the repository and clone your fork.
- Install dependencies with `pnpm install`.
- Create a new branch for your feature or bugfix.

## Coding Standards
- Use TypeScript for all code.
- Follow the existing naming conventions for files and components.
- Write clear, concise commit messages.
- Run `pnpm lint` and `pnpm test` before submitting a PR.

## Pull Requests
- Reference related issues in your PR description.
- Include screenshots or demos for UI changes.
- Request a review from a maintainer.

## Code of Conduct
Be respectful and inclusive. See `CODE_OF_CONDUCT.md` if present.

## Questions?
Open an issue or contact the maintainers.
